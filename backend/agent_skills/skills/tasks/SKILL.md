# Навык: Задачи

Менеджер дел пользователя в стиле Things. Управляй им через инструменты `mcp__tasks__*`,
а не через markdown-файлы.

Инструменты: `create_task`, `list_tasks`, `update_task`, `complete_task`, `delete_task`,
`list_projects`, `create_project`.

Правила работы:
- Когда просят добавить дело, напомнить, запланировать — создавай задачу (`create_task`).
- Поле `when`: `'today'`, `'someday'` либо ISO-дата `'YYYY-MM-DD'`. «Завтра», «в пятницу»
  и т.п. сам переводи в конкретную дату. `deadline` — отдельная ISO-дата (жёсткий срок).
- Группируй по проектам: если дело относится к проекту — указывай `project` (имя; новый
  проект создаётся автоматически).
- Прежде чем менять/закрывать — при необходимости найди задачу через `list_tasks`
  (view: inbox|today|upcoming|anytime|someday|logbook) и работай по её `id`.
- Показывая список, будь лаконичен: группируй по дате/проекту, не вываливай служебные поля.

Если запрос на самом деле про заметки/знания — переключись на навык `wiki`. Если нужно
«сработать в момент времени и прислать» (напоминание по расписанию) — это крон
(`mcp__cron__create_job`), а не задача.
